# Cross-Domain Retrieval Baseline

This baseline is the acceptance reference for general retrieval changes. It
uses fixed, hand-curated development and holdout datasets generated by
`scripts/generate_cross_domain_eval.py`. The generator verifies that every
non-no-result target and acceptable alternative appears in both the 2k and
10k corpora, while every no-result sentinel appears in neither corpus.

Each split contains 60 distinct queries: 12 each for AI/LLM, web, devtools,
infrastructure, and data. Every split includes exact-name, functional,
ecosystem, ambiguous, and no-result queries in both English and Chinese.
Development and holdout queries are disjoint.

## 2k Baseline

The reports are generated, ignored artifacts:

```text
xists eval run --cases data/scale-cross-domain-dev.json --index data/scale-2k-index.json --output data/scale-2k-cross-domain-dev-baseline.json --top-k 10
xists eval run --cases data/scale-cross-domain-holdout.json --index data/scale-2k-index.json --output data/scale-2k-cross-domain-holdout-baseline.json --top-k 10
```

| Split | Recall@1 | Recall@5 | Wrong high-confidence | No-result abstain |

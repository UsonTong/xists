# v0.6.0 Completion Report

Date: 2026-07-19

## Completed

- T1: `profile refresh` writes successful records to `<output>.partial.jsonl`, supports `--resume`, ignores a truncated final line, rejects accidental overwrite, preserves input order, and atomically writes the completed JSON output.
- T2: `ingest github` and `profile refresh` support non-destructive `--dry-run` reports with process/skip counts and estimated calls in text or JSON.
- T3: both batch workflows isolate individual failures, write actionable reports, and support `--retry-failed`; a completed batch with individual failures exits successfully.
- T4: GitHub rate-limit exhaustion rotates tokens or waits until reset within `--max-rate-limit-wait`. Ingest checkpoints now use the same append-only JSONL approach as profile refresh in single-threaded, multi-threaded, and GraphQL-batch paths. `--resume` skips checkpointed records and retains a truncated final checkpoint line for retry.
- T5: eval reports and inspection include `recall@1` and `recall@5`, including declared acceptable alternatives and Chinese cases.
- T6: `docs/scaling-experiment.md` defines the 2k, 10k, and 20k corpus recipe, command sequence, record template, and promotion criteria.

The profile request default timeout is 600 seconds. This is an operational reliability change for local or heavily loaded LLM endpoints; it is covered by a unit test and documented in `docs/usage.md`.
